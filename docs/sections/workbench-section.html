System Architecture Overview

This document outlines the overall system architecture for TankerMade, including component relationships, data flow, and technical decisions.

Architecture Principles

  • Separation of Concerns: Clear separation between presentation, business logic, and data layers
  • Dependency Injection: Loose coupling through dependency inversion
  • Event-Driven: Asynchronous processing where appropriate
  • Testability: Architecture supports unit and integration testing

System Components

The system is organized into the following major components:

Core Domain

Contains business entities, value objects, and domain services. No dependencies on external concerns.

Application Layer

Orchestrates domain operations, handles commands and queries, manages transactions.

Infrastructure

Data persistence, external service integrations, cross-cutting concerns.

Presentation

Blazor components, UI logic, user interaction handling.
