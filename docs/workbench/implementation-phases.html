Implementation Phases

21-Week Development Timeline

Complete roadmap from foundation to production-ready MVP with 10 distinct phases and clear milestones based on the comprehensive project specification.

1

Foundation

Weeks 1-3
Goal: Get basic infrastructure running in both offline and server modes

Deliverables

Solution Structure
  • All projects created
  • Core domain models in TankerMade.Core
  • Service contracts in TankerMade.Contracts
Client Application
  • Blazor WASM client with routing
  • IndexedDB wrapper for offline mode
  • Service injection (offline vs. API based on config)
Server Foundation
  • ASP.NET Core API with minimal endpoints
  • EF Core DbContext and initial migration
  • JWT authentication setup

✅ Acceptance Criteria

  • App runs in offline mode (IndexedDB storage)
  • App can connect to API server with auth
  • User can switch between modes in settings
  • Basic CRUD works for one entity (Projects) in both modes
  • User registration and login functional
  • Role-based authorization working (viewer/editor/admin)
2

Core Features

Weeks 4-6
Goal: Implement all CRUD operations and UI for main entities

Deliverables

Projects & Patterns
  • Full CRUD with wizard, detail view, list with filters
  • Multi-step creation wizards
  • Pattern pieces and steps management
Inventory System
  • Yarn: Full CRUD with lot tracking, purchase history
  • Tools: Full CRUD with size variants
  • Notions: Full CRUD with color/size variants
Supporting Features
  • Kits: Full CRUD with pieces and supplies
  • Reference Data: CRUD UI for all reference tables
  • Project-Inventory Links: UI for linking supplies

✅ Acceptance Criteria

  • All entities have working list, detail, create, and edit pages
  • Multi-step wizards work for Project, Pattern, and Kit creation
  • "Inline Add New" in dropdowns creates reference data without navigation
  • Filters and sorting work on all list pages
  • Search works across entities
  • Slugs generate automatically from names
  • All operations work in both offline and server modes
  • Data validates correctly (required fields, format checks)

Pattern Details & Project Workspace

Weeks 7-8
Goal: Rich pattern editing and project progress tracking

Deliverables

Pattern Management
  • Pattern Pieces: Reorder with drag-drop, inline add/edit/delete
  • Pattern Steps: Inline editing with stitch counts
  • Repeat notation support (e.g., "5-7")
  • Pattern Detail Page: View full pattern with all pieces/steps
Project Workspace
  • Step checklist with checkboxes
  • Piece selector (if pattern has multiple pieces)
  • Progress bar (completion percentage)
  • Stitch counter (completed/total)
Validation System
  • Ensure pieces have steps
  • Warn if total stitches don't match step totals
  • Prevent saving incomplete patterns

✅ Acceptance Criteria

  • User can create complex multi-piece patterns
  • Steps can be reordered within pieces
  • Pieces can be reordered within patterns
  • Project workspace shows current pattern with checkboxes
  • Checking steps updates completion percentage in real-time
  • Stitch counts aggregate correctly
  • Validation prevents saving incomplete patterns

Deliverables

Session Management
  • Session History: View past work sessions

Statistics

  • Average session length

✅ Acceptance Criteria

  • Timer shows in header/nav when active (global visibility)

⚠️ Risk Mitigation Strategies

  • Scope Creep: Strict phase boundaries, clear "Won't do" list, time-boxed features
  • Technical Risk: IndexedDB fallback to localStorage, server-wins conflict resolution
  • Performance: Client-side pagination, lazy loading, archival features
  • Motivation: Small achievable milestones, dogfood the app, build desired features first
